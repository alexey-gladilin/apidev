# Решения и компромиссы

## DEC-01: Разделить output-контуры, но оставить общую модель безопасности
- Решение: `scaffold_dir` независим от `generated_dir`, но оба ограничены единым path-boundary policy.
- Почему: нужна функциональная независимость без роста риска произвольной записи.
- Компромисс: часть «гибких» пользовательских путей останется недопустимой из-за security-ограничений.

## DEC-02: Ввести явную политику записи scaffold
- Решение: `scaffold_write_policy` с режимами `create-missing`, `skip-existing`, `fail-on-conflict`.
- Уточнение: default-значение `scaffold_write_policy` — `create-missing` для сохранения текущего поведения без новых опций.
- Почему: снимает неявность текущего поведения и делает сценарии CI/локальной разработки предсказуемыми.
- Компромисс: пользователю нужно явно выбирать режим для нестандартных workflow.

## DEC-03: Проецировать runtime routing и OpenAPI из единой нормализованной модели
- Решение: runtime adapter и openapi adapter работают от общего operation metadata слоя.
- Почему: исключает расхождение между runtime маршрутами и документированием.
- Компромисс: повышается требовательность к качеству нормализации входных данных.

## DEC-04: Поддержать short-form примера ошибок через канонизацию
- Решение: `errors[].example` принимается и преобразуется в каноническую форму до генерации.
- Почему: снижает шум в контрактах и повышает удобство авторов.
- Компромисс: добавляется новая ветка валидации конфликтов short-form и nested-form.

## DEC-05: Профилизация `init` как декларативный слой
- Решение: integration profiles в `init` задают runtime/mode/dir декларативно, а не через ручные пост-правки.
- Уточнение: `runtime` ограничен enum `fastapi|none`, `integration-mode` ограничен enum `off|scaffold|full`; `repair` и `force` остаются взаимоисключающими.
- Почему: сокращает вариативность начальной настройки и ошибки bootstrap.
- Компромисс: часть «экзотических» комбинаций профилей будет сознательно недоступна.

## DEC-06: Управлять `x-apidev-*` через отдельный feature toggle
- Решение: `openapi.include_extensions` управляет только расширениями.
- Почему: позволяет интеграцию с инструментами, где vendor extensions нежелательны.
- Компромисс: при выключении уменьшается объем диагностически полезной metadata в OpenAPI-выходе.

## DEC-07: Release-state валидируется строго без миграции
- Решение: отклонять legacy ключи и type mismatch для `release_number` через явную validation error.
- Почему: миграционный контур и обратная совместимость исключены из scope текущего change.
- Компромисс: часть существующих конфигураций потребует ручного исправления перед запуском.

## DEC-08: Fail-fast на неоднозначностях и конфликтных состояниях
- Решение: при конфликте путей, policy, legacy-state или error-example — явная ошибка.
- Почему: безопасность и детерминизм важнее «магического» автовосстановления.
- Компромисс: пользователю чаще требуется явное исправление входных данных.

## DEC-09: Домен как source of truth для Swagger-группировки
- Решение: домен endpoint-а определяется из domain layout операции; Swagger tags вычисляются автоматически из домена.
- Почему: ручной `tags` создает дополнительный канал расхождения и повышает риск недетерминированной группировки.
- Компромисс: ручная тонкая настройка группировки в OpenAPI ограничена; при конфликте manual tag/domain применяется fail-fast.
