# CLI Contract (apidev)

Цель документа: дать проверяемый контракт CLI, который можно применять в PR и code review.

## 1. Канонические команды

Набор верхнего уровня:
- `apidev init`
- `apidev validate`
- `apidev diff`
- `apidev gen`

Правило именования:
- Каноничное имя генерации: `gen`.
- Длинные исторические имена допускаются только как алиасы совместимости.

## 2. Контракт help/UX

Обязательные требования:
- Каждый уровень команды поддерживает `-h` и `--help`.
- Root help содержит секции `Usage`, `Options`, `Commands`.
- У каждой команды есть краткое описание действия в root help.
- Ошибки CLI печатаются без traceback для ожидаемых пользовательских ошибок.

Проверка вручную:
- `apidev -h`
- `apidev init -h`
- `apidev gen -h`

## 3. Контракт совместимости

Правила алиасов:
- При переименовании команда сохраняется как скрытый алиас минимум на 1 минорный релиз.
- В changelog/README фиксируется: каноничное имя + срок удаления алиаса.
- Удаление алиаса возможно только после прохождения периода депрекации.

Текущий статус:
- `apidev gen` — каноничное имя.
- `apidev generate` — compatibility alias.

## 4. Контракт кодов выхода

- `0`: успешное выполнение команды.
- `1`: бизнес-ошибка/валидация/дрейф/невалидный ввод.
- `2`: ошибка парсинга CLI (неизвестный флаг, неверная сигнатура).

## 5. Контракт сообщений

- Тексты успеха и ошибок короткие и однозначные.
- Сообщение ошибки содержит next step (что сделать дальше).
- Формулировки стабильны и пригодны для CI-логов.

## 6. Минимальные тесты для любого изменения CLI

Обязательный набор:
- Тест на `-h` для root.
- Тест на `-h` для минимум одной подкоманды.
- Тест доступности каноничной команды (`gen`).
- Если есть алиас: тест, что алиас вызывает тот же handler/поведение.

Команда запуска:
- `uv run pytest tests/unit/test_cli_conventions.py`

## 7. Чеклист PR (копировать в описание)

- [ ] Добавлены/обновлены help-описания команд.
- [ ] Поддерживаются оба help-флага: `-h` и `--help`.
- [ ] Не сломана каноническая команда `gen`.
- [ ] Алиасы (если есть) задокументированы в README.
- [ ] Обновлены/добавлены CLI-тесты.
