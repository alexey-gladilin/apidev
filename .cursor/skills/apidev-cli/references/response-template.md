# Шаблон ответа агента для apidev-команд

Использовать компактную структуру отчета после запуска `@apidev` или авто-подобранных команд.

## Формат

```markdown
Выполнено:
1. `apidev validate --json`
2. `apidev diff --json`

Результат:
1. [Короткий статус по каждой команде]
2. [Найденные проблемы/изменения]

Требует подтверждения:
1. `apidev gen` (запись изменений в файлы)

Следующий шаг:
1. [Самый безопасный и конкретный]
```

## Правила подачи результата

- Не выводить сырой лог целиком, если нет отдельного запроса.
- Ошибки показывать с конкретной причиной и командой, на которой они возникли.
- Если ошибка указывает на отсутствие команды `apidev`, предлагать/использовать fallback `uv run apidev ...`.
- Для успешных проверок сообщать, что именно проверено (`validate`, `diff`, `gen --check`).
- Для изменений в файловой системе явно помечать, что это write-операция.
